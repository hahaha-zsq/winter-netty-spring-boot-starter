/ 项目简介 概述

将基于 Netty 的 WebSocket 服务零配置地嵌入 Spring Boot 应用的启动器， 一行配置即可运行。

基于 Spring Boot 2.6.11 与 Netty 4.2.7.Final 构建，提供开箱即用的 WebSocket 服务端与 TCP 客户端。仅需一个配置项 — winter-netty.server.websocket.enabled=true — 即可开始接收 WebSocket 连接。零样板代码，零额外依赖。

项目通过 Spring Boot 的 Auto-Configuration 机制，自动创建并管理 Netty 的 EventLoopGroup、ChannelInitializer、业务 Handler 等全部组件。 开发者无需编写任何 Netty 样板代码，只需专注于消息处理逻辑。

为什么选择 Winter Netty
  • 传统方式集成 Netty WebSocket 需要大量样板代码：启动配置、生命周期管理、线程协调、断线重连等。
  • 本 Starter 通过 Auto-Configuration 自动化这一切，让开发者专注于业务而非基础设施。
  • 所有核心扩展点均基于 @ConditionalOnMissingBean，提供默认实现的同时支持一键替换。
  • 适用于实时消息推送、在线协作编辑、直播弹幕、物联网设备通信、实时监控看板等场景。
  • 服务端与客户端共享同一套配置体系，线程池隔离，生命周期独立管理。

/ 核心能力 特性

01 零配置 WebSocket 服务端 一行配置即可启用完整的 WebSocket 服务。自动装配创建并启动 Netty ServerBootstrap，绑定端口并开始接收连接。无需编写 ChannelInitializer 或 EventLoopGroup 代码。 核心
02 Token 认证机制 在 WebSocket 握手阶段拦截请求进行身份认证。支持 URL 参数（?token=）、Authorization: Bearer 头、自定义 Token 头三种提取方式。通过 TokenAuthenticator 接口可插拔替换，认证失败返回 401/403。 安全
03 双向会话管理 维护 userId 到 Channel、channelId 到 userId 的双向映射。基于 ChannelGroup 管理所有在线连接。自动清理断线会话，支持重复登录踢下线 — 同一 userId 新连接会关闭旧连接。 核心
04 消息权限校验 MessagePermissionValidator 接口支持按操作类型（PRIVATE / BROADCAST / SYSTEM）校验用户权限。内置速率限制器（默认 10 条/秒/用户），防止消息洪泛。可扩展为基于角色的访问控制。 安全
05 双套消息推送 API 安全 API（需 Token，经完整鉴权）：sendSystemMessage / sendPrivateMessage / broadcastSystemMessage / broadcastMessage / sendSystemMessageToUsers，均支持同步与 CompletableFuture 异步。内部 API（无需 Token，服务端专用）：sendSystemMessageInternal / broadcastSystemMessageInternal / sendSystemMessageToUsersInternal，适用于定时任务与事件驱动。 API
06 可配置心跳检测 集成 Netty IdleStateHandler，可独立配置读空闲、写空闲、读写全部空闲超时时间。超时后自动关闭连接并清理会话，防止僵尸连接占用资源。 核心
07 CORS 跨域 + 连接限流 内置 CORS 处理器，支持跨域 WebSocket 连接。可配置全局最大连接数（默认 1000），超出返回 503。结合用户级速率限制，从多个维度保障服务稳定性。 基础设施
08 可插拔 Netty 管道 NettyServerPipelineCustomizer 接口允许完全自定义服务端管道 Handler 链。客户端同理支持多个 NettyClientPipelineCustomizer 叠加。WebSocket 模式默认提供完整 8 处理器管道。 扩展
09 独立线程池隔离 服务端与客户端使用独立的 ThreadPoolTaskExecutor。支持分别配置核心线程数、最大线程数、队列容量、线程名前缀、keep-alive 秒数。关闭时可配置是否等待任务完成及超时时间。 基础设施
10 Spring 生命周期管理 @PostConstruct 注解方法在独立线程中异步启动服务，不阻塞 Spring 容器初始化。@PreDestroy 方法进行优雅关闭：先关闭 Channel 再关闭 EventLoopGroup，支持超时回收。 核心
11 消息序列化缓存 SYSTEM 和 BROADCAST 类型消息的 JSON 序列化结果缓存在内存中，相同内容的消息无需重复序列化。内置 totalSent、totalFailed、totalBlocked 统计计数器，可通过 API 查询和重置。 性能
12 TCP 客户端支持 基于 Netty Bootstrap + NioSocketChannel 的 TCP 客户端。可配置目标主机、端口、线程池。支持多个 NettyClientPipelineCustomizer 构建管道。默认关闭，按需启用。 扩展

/ 架构设计 架构

WebSocket 服务端管道

每个 WebSocket 连接建立时，ChannelPipeline 按以下顺序注册 8 个处理器。 标记红色的两个为可扩展节点 —— 开发者可替换认证逻辑与消息处理逻辑。

IdleStateHandler → HttpServerCodec → HttpObjectAggregator → ChunkedWriteHandler → CorsHandler → WebSocketHandshakeAuth → WebSocketProtocol → WebSocketServerHandler

各处理器职责

IdleStateHandler 检测连接空闲状态，触发 IdleStateEvent 供后续处理
HttpServerCodec + Aggregator HTTP 编解码与聚合，将 HttpRequest 聚合为 FullHttpRequest
ChunkedWriteHandler 支持分块写入，大文件或流式数据传输
CorsHandler 处理跨域请求，配置允许的 Origin
WebSocketHandshakeAuth 提取 Token、调用 TokenAuthenticator、连接数检查，握手成功前拦截
WebSocketServerProtocolHandler 处理 WebSocket 升级握手，管理帧类型
WebSocketServerHandler (业务处理器) 处理 HandshakeComplete 事件注册会话。消息分发：HEARTBEAT 回复 PONG，BROADCAST 广播，PRIVATE 路由。channelInactive 清理会话。

组件架构图

Spring Boot 应用 ├═ winter-netty 配置 (application.yml) ├═ @EnableAutoConfiguration (via spring.factories) ├ ├═ NettyServerAutoConfiguration // 条件: enable-server=true ├ └═ NettyClientAutoConfiguration // 条件: enable-client=true ├ ├═ 服务端组件 (WebSocket 启用时) ├ WebSocketSessionManager // 用户 ↔ Channel 双向映射 ├ TokenAuthenticator // 可插拔 Token 认证 ├ MessagePermissionValidator // 可插拔消息权限 ├ WebSocketMessageService // 消息发送 API + 统计 ├ WebSocketServerHandler // @Sharable 业务处理器 ├ NettyServer // boss + worker 线程组 ├ └═ 客户端组件 (TCP 启用时) NettyClient → EventLoopGroup → Bootstrap → connect(host, port)

Bean 加载顺序

Spring 容器启动时，Auto-Configuration 按以下顺序注册 11 个 Bean。 每个步骤的 Bean 都可以被自定义 @Component 替代。

NettyProperties → ThreadPoolExecutor → SessionManager → TokenAuthenticator → PermissionValidator → MessageService → ServerHandler → AuthHandler → PipelineCustomizer → ChannelInitializer → NettyServer

WebSocket 消息生命周期

一条 PRIVATE 消息从发送到接收的完整链路：

步骤 1 客户端发送 JSON 消息帧
步骤 2 WebSocketServerHandler.channelRead0 反序列化为 NettyMessage 对象
步骤 3 按 type 分发：HEARTBEAT / SYSTEM / BROADCAST / PRIVATE
步骤 4 TokenAuthenticator 校验发送方身份
步骤 5 MessagePermissionValidator 校验操作权限 + 速率限制检查
步骤 6 消息长度校验 + JSON 序列化（命中缓存则跳过）
步骤 7 通过 SessionManager 获取目标 Channel，writeAndFlush 发送
步骤 8 更新统计计数器（totalSent / totalFailed / totalBlocked）

配置场景

仅 WebSocket 服务端 winter-netty.server.websocket.enabled=true 完整的 WebSocket 栈：认证、会话管理、消息收发、心跳检测、CORS
仅 TCP 服务端 websocket.enabled=false + 自定义 PipelineCustomizer 裸 TCP 模式，通过 NettyServerPipelineCustomizer 完全自定义 Handler 链
仅客户端模式 enable-server=false, enable-client=true 独立 TCP 客户端，支持多个 NettyClientPipelineCustomizer 叠加构建管道
服务端 + 客户端 两者均启用 独立线程池、隔离生命周期、共享 NettyProperties 配置空间

/ 使用流程 工作流程

01
添加 Maven 依赖
在项目的 pom.xml 中引入以下依赖。已发布至 Maven 中央仓库， 无需额外配置仓库地址。
<!-- pom.xml --> <dependency> <groupId>io.github.hahaha-zsq</groupId> <artifactId>winter-netty-spring-boot-starter</artifactId> <version>0.0.6</version> </dependency>
02
配置 application.yml
在配置文件中启用 WebSocket 服务。所有配置项均以 winter-netty 为前缀， 支持 IDE 自动补全（已提供 spring-configuration-metadata.json）。
# application.yml — 完整配置参考 winter-netty: # 服务端配置 server: port: 8888 boss-threads: 1 worker-threads: 4 websocket: enabled: true path: "/ws" heartbeat-interval: 30 max-connections: 1000 thread-pool: core-pool-size: 2 max-pool-size: 4 queue-capacity: 100 name-prefix: "winter-server-"
03
启动 Spring Boot 应用
直接运行 Spring Boot 启动类。Auto-Configuration 自动检测依赖与配置， 创建所有必要的 Bean。Netty 服务端通过 @PostConstruct 在 winterNettyServerTaskExecutor 独立线程池中异步启动， 不会阻塞主线程。
// 启动日志示例 [ winter-netty ] Starting Netty server on port 8888... [ winter-netty ] WebSocket pipeline initialized: idle → http → auth → ws → handler [ winter-netty ] Server started successfully, listening on /0:0:0:0:0:0:0:0:8888
04
客户端建立 WebSocket 连接
客户端通过标准 WebSocket 协议连接至服务端。握手阶段自动完成 Token 提取与认证。 认证通过后升级为 WebSocket 长连接，SessionManager 注册用户与 Channel 的映射关系。
// JavaScript 客户端示例 const ws = new WebSocket("ws://localhost:8888/ws?token=xxx"); ws.onopen = () => console.log("[CONNECTED]"); ws.onmessage = (e) => { const msg = JSON.parse(e.data); // { type, fromUserId, content, timestamp } }; ws.onclose = () => console.log("[DISCONNECTED]");
05
消息收发与 REST API 推送
客户端可发送 HEARTBEAT（心跳）、BROADCAST（广播）、PRIVATE（私信）消息。 服务端可通过注入 WebSocketMessageService 在 REST 控制器中 主动推送 SYSTEM 消息。
// REST 控制器示例 — 服务端主动推送 @RestController @RequestMapping("/api/message") public class PushController { @Autowired private WebSocketMessageService messageService; @PostMapping("/system/send") public Result sendSystem( @RequestParam String userId, @RequestParam String content) { messageService.sendSystemMessage(userId, content); return Result.ok("[SENT]"); } @PostMapping("/system/broadcast") public Result broadcast(@RequestBody String content) { messageService.broadcastSystemMessage(content); return Result.ok("[BROADCAST]"); } @GetMapping("/online/count") public long onlineCount() { return messageService.getOnlineCount(); } }
06
断开连接与优雅关闭
客户端断线时，channelInactive 触发， SessionManager 自动移除 userId 与 Channel 的映射。应用关闭时， @PreDestroy 方法先关闭所有连接， 再调用 shutdownGracefully(5, 10, TimeUnit.SECONDS) 释放 EventLoopGroup 线程资源。
// 关闭日志示例 [ winter-netty ] Shutting down Netty server... [ winter-netty ] Closing 12 active channels... [ winter-netty ] EventLoopGroup shutdown complete [ winter-netty ] Server stopped

/ 可扩展处 扩展点

所有扩展点均基于 @ConditionalOnMissingBean 实现。 创建实现接口的 @Component Bean， 即可自动替换默认实现，无需修改框架代码。

接口 默认实现 用途与扩展方式
TokenAuthenticator DefaultTokenAuthenticator WebSocket 握手时校验 Token 并返回 userId。默认仅在开发环境解析 "user:xxx" 格式。可替换为 JWT / OAuth2 / 自定义 Token 校验。
MessagePermissionValidator DefaultMessagePermissionValidator 按操作类型校验用户权限，内置 RateLimiter（10 条/秒）。可扩展为基于角色的访问控制（RBAC）或自定义限流策略。
NettyServerPipelineCustomizer WebSocketPipelineCustomizer 自定义服务端 ChannelPipeline。WebSocket 模式下可继承 WebSocketPipelineCustomizer 扩展；TCP 模式下需完全自定义处理器链。
NettyClientPipelineCustomizer 无（必须提供） 构建客户端管道。支持多个 @Component 实现自动叠加，按集合注入。List<NettyClientPipelineCustomizer>
WebSocketServerHandler WebSocketServerHandler @Sharable 单例业务处理器。继承并重写 channelRead0 可添加消息持久化（存入数据库 / Redis）、消息转换、自定义路由逻辑。
WebSocketSessionManager WebSocketSessionManager 用户与 Channel 的双向映射注册表。扩展可添加集群支持（Redis 共享会话）、自定义事件钩子、查询增强。
WebSocketMessageService WebSocketMessageService 消息发送 API 外观层。扩展可添加消息加密、投递确认重试机制、消息轨迹追踪或自定义 Telemetry。

典型扩展示例：JWT Token 认证

替换 DefaultTokenAuthenticator 只需创建一个 @Component 实现 TokenAuthenticator 接口：

// 自定义 JWT Token 认证器 — 自动替换默认实现 @Component public class JwtTokenAuthenticator implements TokenAuthenticator { @Override public AuthResult authenticate(String token) { try { // 解析 JWT Token String userId = JWT.decode(token) .getClaim("sub").asString(); return AuthResult.success(userId); } catch (Exception e) { return AuthResult.fail("invalid token"); } } }

典型扩展示例：消息持久化

继承 WebSocketServerHandler 即可在消息处理前后 添加自定义逻辑，如保存到数据库：

// 消息持久化处理器 — 继承默认处理器 @Component public class PersistentWebSocketHandler extends WebSocketServerHandler { @Autowired private MessageRepository messageRepo; @Override protected void handlePrivateMessage(ChannelHandlerContext ctx, NettyMessage msg) { // 持久化消息后调用父类发送 messageRepo.save(msg); super.handlePrivateMessage(ctx, msg); } }

完整可替换 Bean 清单

以下 Bean 均支持通过 @Component 或 @Bean 替换：

/ 消息推送能力 消息 API

WebSocketMessageService 提供两套消息推送接口： 安全 API（需要 Token，经过完整鉴权流程）与 内部 API（无需 Token，服务端主动推送专用）。

功能矩阵

方法 说明 Token 异步
sendSystemMessage 发送系统消息给指定用户 ✓ ✓
sendPrivateMessage 发送私聊消息 ✓ ✓
broadcastSystemMessage 广播系统消息给所有在线用户 ✓ ✓
broadcastMessage 广播普通消息（排除发送者） ✓ —
sendSystemMessageToUsers 批量发送系统消息 ✓ —
sendSystemMessageInternal 内部推送系统消息（无需 Token） — ✓
broadcastSystemMessageInternal 内部广播系统消息（无需 Token） — ✓
sendSystemMessageToUsersInternal 内部批量发送（无需 Token） — —

安全 API 鉴权流程

每次调用安全 API 时，validateSecurity() 依次执行四步校验：

Token 验证 → 身份认证 → 在线检查 → 频率限制 → 权限校验
安全 API 适用场景：用户触发的操作（前端调用 REST 接口发送消息）。需要管理员 Token 或发送者 Token，经过完整鉴权。
内部 API 适用场景：定时任务、事件驱动、微服务回调。无需 Token，跳过在线检查和频率限制，直接投递。

安全 API 示例

// REST 控制器 — 需要管理员 Token @PostMapping("/system/send") public Result sendSystem( @RequestParam String adminToken, @RequestParam String userId, @RequestParam String content) { return Result.ok(messageService.sendSystemMessage(adminToken, userId, content)); } // 私聊消息 — 需要发送者 Token @PostMapping("/private/send") public Result sendPrivate( @RequestParam String senderToken, @RequestParam String toUserId, @RequestParam String content) { return Result.ok(messageService.sendPrivateMessage(senderToken, toUserId, content)); } // 批量发送 — 管理员 Token messageService.sendSystemMessageToUsers(adminToken, userIds, content); // 异步发送 messageService.sendSystemMessageAsync(adminToken, userId, content) .thenAccept(result -> log.info("[SENT] success={}", result.isSuccess()));

内部 API 示例（服务端主动推送）

// 定时任务 — 每日早报推送 @Scheduled(cron = "0 0 9 * * ?") public void dailyReminder() { messageService.broadcastSystemMessageInternal("早上好！今日待办事项已更新"); } // 事件驱动 — 业务事件触发推送 @EventListener public void onOrderCreated(OrderCreatedEvent event) { messageService.sendSystemMessageInternal( event.getCustomerId(), "您的订单已创建: " + event.getOrderId() ); } // 内部异步推送 messageService.sendSystemMessageAsyncInternal(userId, content) .thenAccept(result -> { if (result.isSuccess()) log.info("[INTERNAL] 投递成功"); }); // 内部批量推送 messageService.sendSystemMessageToUsersInternal( Arrays.asList("user1", "user2", "user3"), "恭喜您获得 VIP 资格" );

返回值结构

SendResult isSuccess() 是否成功 getErrorMessage() 失败原因
BroadcastResult isSuccess() 是否成功 getSuccessCount() 成功用户数
BatchSendResult getSuccessCount() 成功数 getFailedCount() 失败数 getOfflineCount() 离线数
